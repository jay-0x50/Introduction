// "보이지 않는 로직이 플레이어의 손끝에 전달되는 순간을 설계합니다"

저는 게임 안에서 일어나는 모든 현상 뒤의 수식과 구조에 집착하는 개발자입니다. 초등학교 3학년 때 스크래치로 처음 프로그래밍을 접한 이후, 중학교 시절 오렌지보드로 자동 손 소독제 기계를 직접 제작해 학급에 기부하는 경험을 했습니다. 이 과정에서 "코드가 현실에 닿는 순간"의 가치를 체감했고, 그 감각을 가장 직접적으로 구현할 수 있는 영역이 게임플레이 프로그래밍임을 확신하게 되었습니다.

게임을 플레이할 때마다 저는 항상 "이 타격감의 수식은 무엇인가", "이 판정 로직은 어떤 구조로 동작하는가"를 분해해 왔습니다. 그 호기심은 자연스럽게 실제 구현으로 이어져, 학창 시절 동안 리듬 게임의 ms 단위 판정 시스템부터 엔진 없는 3D 렌더링 파이프라인까지 2,300시간 이상, 13개 프로젝트의 코어 로직을 직접 설계했습니다.

저의 목표는 기획자의 의도를 기술적 제약 없이 플레이 경험으로 정확히 변환하는 것입니다. 보이지 않는 구조를 안정적으로 설계하고, 플레이어가 화면을 통해 정확히 느끼게 만드는 게임플레이 프로그래머로서 즉시 기여하겠습니다.

프로젝트
경험
// "문제를 정의하고, 구조로 해결하고, 수치로 검증했습니다"

PyMax (2D 리듬 게임, Python/Pygame)에서는 노트 스폰 → 입력 감지 → 판정 처리 → 시각 피드백의 전체 파이프라인을 설계했습니다. 초기 구현에서 발생한 사운드-노트 간 싱크 오프셋 문제를 프레임 기반 타이머에서 시스템 클럭 기반 타이머로 교체해 해결했고, smoothscale 최적화와 Object Pooling으로 프레임 드랍을 완전히 제거해 60fps 안정성을 확보했습니다.

SpaceOut (DirectX 11 3D 슈팅)에서는 외부 엔진 없이 뷰·프로젝션 행렬과 렌더링 파이프라인을 직접 구축했습니다. "엔진이 없으면 3D 게임을 만들 수 없다"는 제약을 직접 해제한 경험으로, 어떤 환경에서도 플레이 로직을 구조화할 수 있는 기초 역량을 증명했습니다. AABB 충돌 수식과 Object Pool + 캐시 친화적 자료구조를 결합해 프레임 히치를 측정 후 제거했습니다.

팀 프로젝트 '마법소녀카와이러블리즈' (C++, 6인)에서는 메인 프로그래머로서 보스 레이드 코어 로직과 마이크 음성 인식 서버 연동을 전담했습니다. GitHub Flow 기반의 브랜치 전략을 직접 수립하고, 코드 리뷰와 트러블슈팅 대응으로 팀 전체의 개발 속도와 안정성을 유지했습니다.

자기소개서 박재영
강점 &
협업
// "팀의 빈틈을 찾아 메우는 것이 제 역할입니다"

저는 맡은 영역을 완전히 책임지고, 완료 이후에는 팀 전체의 속도를 높이는 방향으로 움직입니다. 'CodeUs' 동아리 부장으로서, 그리고 다수의 팀 프로젝트에서 메인 프로그래머를 맡으며 일정과 품질을 동시에 지켜내는 습관을 만들었습니다.

제 업무를 완료한 이후에는 항상 팀원의 블로킹 이슈를 함께 분석했습니다. 예상치 못한 버그 앞에서 "왜 이 시점에, 이 코드에서 터지는가"를 역추적하는 디버깅을 즐기며, 실제로 여러 프로젝트에서 배포 직전의 크리티컬 이슈를 팀원과 함께 해결한 경험이 있습니다. 코드 리뷰는 완성된 코드를 검토하는 것이 아니라, 설계 방향을 함께 정렬하는 과정이라고 생각합니다.

교내 배포(50~70명 대상)에서 직접 피드백을 수집하고 이를 코드 수정으로 연결하는 경험을 반복하면서, 개발자가 사용자와 가장 가까이 있어야 한다는 관점이 생겼습니다. 이 관점이 더 나은 플레이 경험을 만드는 원동력이 되고 있습니다.

포부

신입이지만 13개 프로젝트에서 코어 로직을 직접 설계하고, 성능 문제를 수치로 검증하며, 팀 협업을 주도한 경험이 있습니다. 새로운 코드베이스에 적응하는 것보다, 지금 팀이 해결하지 못한 기술 문제를 찾아 해결하는 것에 더 관심이 있습니다.

입사 후 단기 목표는 프로젝트의 플레이 루프에서 가장 성능 민감한 구간을 파악하고, 측정 가능한 개선을 만드는 것입니다. 중기 목표는 기획 의도를 기술 제약 없이 구현할 수 있는 시스템 구조를 제안하는 개발자로 성장하는 것입니다. 안정적이고 확장 가능한 코드가 플레이어가 반복해서 손이 가는 재미를 만드는 기반이라고 믿습니다.
