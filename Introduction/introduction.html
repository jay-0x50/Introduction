개발을 '재미'로 번역하는 게임 프로그래머

어린 시절부터 저는 게임을 즐길 때마다 "이 통쾌한 타격감은 어떤 수식으로 만들어졌을까?",
"이 절묘한 판정은 어떤 규칙으로 돌아갈까?"라는 호기심을 품어왔습니다.
그 호기심은 자연스레 기획서 상의 아이디어를 화면 속에서 살아 움직이는
'플레이'로 구현해 내는 게임 개발의 길로 저를 이끌었습니다.

저에게 코딩은 단순한 기능 구현이 아니라, 플레이어가 느끼는 '찰나의 즐거움'을 설계하는 작업입니다.
보이지 않는 논리들을 정교하게 맞물리게 하여 유저에게 최고의 몰입감을 선사하는
게임 프로그래머가 되고자 지원합니다.

장르를 넘나들며 다진 '플레이 로직' 구현 역량

10여 개의 프로젝트를 진행하며 게임이 동작하는 코어 로직을 집요하게 탐구했습니다.
2D 리듬 게임인 PyMax 개발 당시,
단순한 노트 드롭을 넘어 마이크로초 단위의 정확한 판정 시스템을 설계했습니다.
유저의 입력과 시각적 피드백(애니메이션) 간의 오차를 줄여 조작감을 극대화하는 과정을 통해
'게임의 손맛'을살리는 최적화를 경험했습니다.

또한, SpaceOut (슈팅 게임) 프로젝트에서는 DirectX 3D를 활용하여 3D 공간에서의 게임 로직을
직접 구현했습니다.엔진에 의존하지 않고 뷰/프로젝션 행렬 변환과 Bounding Box 알고리즘을 수학적으로
계산하여 정밀한 충돌 처리 로직을 완성하며, 어떤 환경에서도 게임의 규칙을 구현할 수 있는
선형대수학 지식과 렌더링 파이프라인의 이해도를 증명했습니다.

맡은 바를 완수하고 팀의 빈틈을 채우는 책임감

팀 프로젝트에서 가장 기본적이고 중요한 협업은 '내가 맡은 역할을 끝까지 책임지고 완수하는 것'이라고 생각합니다.
저는 전공 동아리 'CodeUs'와 다수의 프로젝트에서 메인 프로그래머를 맡으며, 누군가를 지시하고 이끌기보다는
제게 주어진 핵심 로직과 시스템 구현을 버그 없이 확실하게 책임지는 실무자로서 임했습니다.

제 업무를 안정적으로 마무리한 후에는 늘 팀원들의 상황을 살폈습니다.
해커톤처럼 기한이 촉박한 환경에서 예상치 못한 오류나 복잡한 로직으로 막힌 동료가 있으면,
주저 없이 함께 코드를 보며 디버깅을 돕고 해결책을 찾았습니다.
스스로 팀의 빈틈을 찾아 메우며, 프로젝트가 무사히 완성될 수 있도록 묵묵히 지원하는
든든한 동료이자 조력자가 되겠습니다.

플레이어의 경험을 최우선으로 생각하는 개발자

저의 목표는 기획의 참신함을 기술적 제약 없이 구현해 내고,
끊임없이 '더 나은 조작감과 재미'를 고민하는 전천후 게임 프로그래머로 성장하는 것입니다.

지금까지 다양한 장르의 메인 로직을 구현하고 시스템 기저의 원리를
파악하며 쌓아온 탄탄한 기본기를 바탕으로,
팀의 프로젝트에 빠르게 녹아들겠습니다.
당장 눈앞의 버그를 고치는 것에 급급하지 않고,
최적화된 코드와 유연한 구조로 플레이어에게 최고의 경험을 제공하는 핵심 인재가 되겠습니다.
