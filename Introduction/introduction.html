기획을 '재미'로 번역하는 게임 프로그래머

어린 시절부터 저는 게임을 즐길 때마다 "이 통쾌한 타격감은 어떤 수식으로 만들어졌을까?", "이 절묘한 판정은 어떤 규칙으로 돌아갈까?"라는 호기심을 품어왔습니다. 그 호기심은 자연스레 기획서 상의 아이디어를 화면 속에서 살아 움직이는 '플레이'로 구현해 내는 게임 개발의 길로 저를 이끌었습니다.

저에게 코딩은 단순한 기능 구현이 아니라, 플레이어가 느끼는 '찰나의 즐거움'을 설계하는 작업입니다. 보이지 않는 논리들을 정교하게 맞물리게 하여 유저에게 최고의 몰입감을 선사하는 게임 프로그래머가 되고자 지원합니다.

장르를 넘나들며 다진 '플레이 로직' 구현 역량

10여 개의 프로젝트를 진행하며 게임이 동작하는 코어 로직을 집요하게 탐구했습니다. 2D 리듬 게임인 PyMax 개발 당시, 단순한 노트 드롭을 넘어 마이크로초 단위의 정확한 판정 시스템을 설계했습니다. 유저의 입력과 시각적 피드백(애니메이션) 간의 오차를 줄여 조작감을 극대화하는 과정을 통해 '게임의 손맛'을 살리는 최적화를 경험했습니다.

또한, SpaceOut (슈팅 게임) 프로젝트에서는 DirectX 3D를 활용하여 3D 공간에서의 게임 로직을 직접 구현했습니다. 엔진에 의존하지 않고 뷰/프로젝션 행렬 변환과 Bounding Box 알고리즘을 수학적으로 계산하여 정밀한 충돌 처리 로직을 완성하며, 어떤 환경에서도 게임의 규칙을 구현할 수 있는 선형대수학 지식과 렌더링 파이프라인의 이해도를 증명했습니다.

기획과 아트를 기술로 엮어내는 실무형 리더십

게임 개발은 프로그래머 혼자 하는 것이 아닙니다. 기획자의 의도와 아티스트의 리소스를 코드로 결합하는 최전선에 있는 만큼, 소통과 조율이 가장 중요하다고 믿습니다. 저는 전공 동아리 'CodeUs'의 부장 및 다수의 프로젝트에서 메인 프로그래머를 맡으며 팀원들의 역량을 파악해 적재적소에 역할을 분배하는 조율자 역할을 수행했습니다.

해커톤과 같은 제한된 시간 속에서도 기획의 독특한 아이디어(마우스 드래그 기반 조합, 단어 학습에 따른 성장 등)를 구체적인 게임 로직으로 빠르게 프로토타이핑해 냈습니다. 누군가 어려움을 겪을 때는 함께 코드를 보며 해결책을 찾고, 프로젝트가 무사히 완성되도록 전체적인 뼈대를 잡는 든든한 페이스 메이커 역할을 자신합니다.

플레이어의 경험을 최우선으로 생각하는 개발자

저의 목표는 기획의 참신함을 기술적 제약 없이 구현해 내고, 끊임없이 '더 나은 조작감과 재미'를 고민하는 전천후 게임 프로그래머로 성장하는 것입니다.

지금까지 다양한 장르의 메인 로직을 구현하고 시스템 기저의 원리를 파악하며 쌓아온 탄탄한 기본기를 바탕으로, 팀의 프로젝트에 빠르게 녹아들겠습니다. 당장 눈앞의 버그를 고치는 것에 급급하지 않고, 최적화된 코드와 유연한 구조로 플레이어에게 최고의 경험을 제공하는 핵심 인재가 되겠습니다.
