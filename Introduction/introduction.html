엔진의 심장을 설계하는 게임 플레이 개발자

처음 게임에 빠진 건 플레이어로서였지만, 어느 순간부터 제 관심사는
"이 물리 판정과 렌더링은 엔진 내부에서 어떻게 처리될까?"로 옮겨갔습니다.
단순히 제공되는 툴을 사용하는 것에 만족하지 못하고
C와 Python을 활용해 리듬 게임을 바닥부터 만들어보면서,
제가 진정으로 원하는 것은 게임의 겉모습이 아닌
동작 원리를 직접 설계하고 엔진을 통제하는 것임을 깨달았습니다.

이러한 기술적 갈증은 자연스럽게 게임 엔진의 기저를 파고드는 계기가 되었습니다.
C++와 DirectX 11로 렌더링 파이프라인을 직접 구현해보고, C#과 Unity, Unreal Engine 5를 다루며
프레임워크가 내부적으로 메모리와 게임 루프를 어떻게 관리하는지 집요하게 탐구해 왔습니다.
미림마이스터고의 개발 동아리 'CodeUs'의 부장으로서, 저는 늘 팀원들에게
'왜 이렇게 동작하는가'라는 근본적인 질문을 던지며 기술적 깊이를 더하는 데 집중했습니다.

클라이언트와 서버, 보안을 아우르는 넒은 시야

가장 애착이 가는 프로젝트인 Python 기반 리듬 게임 PyMax에서는 노트가 쌓일수록 발생하는 프레임 드랍을
smoothscale 최적화와 딕셔너리 캐싱을 통해 드로우콜을 최소화하여 해결했습니다.
성능 최적화가 곧 플레이어의 쾌적한 경험(UX)으로 직결된다는 것을 체감한 순간이었습니다.

저의 탐구심은 클라이언트에만 머물지 않았습니다. 다수의 플레이어가 상호작용하는 멀티플레이 환경과
게임 서버 인프라를 깊이 이해하기 위해 Java, Spring, React, JS를 활용한 풀스택 환경을 구축해 보았습니다.
또한, PyRansomware 프로젝트를 통해 Kali Linux 환경에서 원격 제어 및 네트워크 보안을 테스트하며,
안전한 서버 아키텍처와 데이터 무결성에 대한 감각을 키웠습니다.
이러한 폭넓은 경험은 현재 Unreal Engine 5와 Unity에서 네트워크 동기화나 최적화된 로직을 설계할 때
강력한 구조적 밑거름이 되고 있습니다.

책임감으로 아키텍처의 빈틈을 채우는 협업 스타일

저는 견고한 코드로 플레이어의 재미를 책임지는 사람입니다.
게임 기획이 의도한 타격감과 조작감을 프레임 드랍 없는 최적의 로직으로 깎아내는 과정을 사랑합니다.

협업에 있어서는 팀의 목표 달성을 위해 시스템의 빈틈을 찾아 메우는 것을 최우선으로 여깁니다.
Visual Studio와 VS Code 환경에서 Git, GitHub를 활용해 체계적인 버전 관리와 브랜치 전략을 수립하고,
동아리 리더로서 코드 리뷰 문화를 주도했습니다.
엔진 내부의 한계에 부딪히거나 서버 연동에 문제가 생기면, 언어나 툴을 가리지 않고
공식 문서와 로우 레벨 코드를 분석하여 주도적으로 해결책을 찾아내는 든든한 실무자입니다.

앞으로의 목표: 한계를 넘어서는 엔진 & 서버 개발자

저의 최종 목표는 단순히 상용 엔진을 사용하는 것을 넘어,
엔진의 핵심 기능을 직접 확장하고 최적화하는 엔진 개발자이자
클라이언트와 서버의 통신 병목을 해결하는 멀티플레이 서버 개발자로 성장하는 것입니다.

현재는 Unreal Engine 5의 C++ 프레임워크와 멀티스레딩 최적화, GAS(Gameplay Ability System)를 활용한
확장성 있는 아키텍처 설계에 집중하고 있습니다. 앞으로 자체 물리 연산 최적화, 렌더링 엔진 고도화,
그리고 대규모 트래픽을 처리하는 분산 서버 설계까지 영역을 넓혀,
어떤 기술적 난관이든 뚫고 나가는 압도적인 퍼포먼스의 게임 플레이 개발자가 되겠습니다.
